- **Thorough Verification**: Agy (Antigravity) must double-check work before implementing changes. This means deeply scanning through all relevant files and being completely certain before making any modifications.

Use the connection string `BettingAppDB_Dev` found in `dotnet user-secrets list`.
- **CRITICAL: Editing Production Data**: Reading PROD data is fine, but if the user ever asks to EDIT, UPDATE, or DELETE any production data, you MUST triple-check with the user first. Before making any destructive or modifying queries, you MUST explicitly provide the user with the exact data that will be changed and the exact script that will run, and wait for their explicit confirmation before executing.
